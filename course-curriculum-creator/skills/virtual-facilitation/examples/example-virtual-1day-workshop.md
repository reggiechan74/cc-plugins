# Example Virtual Adaptation: 1-Day Project Management Workshop

**Course:** Introduction to Agile Project Management
**Duration:** 1 day virtual (2 sessions Ã— 3.5 hours, 9:00-12:30 and 13:30-16:30)
**Audience:** Team leads and managers new to Agile; 15-20 participants
**Platform:** Zoom with breakout rooms, Miro for whiteboarding, Google Docs for collaboration
**Original Format:** In-person 6-hour workshop

## In-Person vs. Virtual Comparison

| Aspect | Original In-Person | Virtual Adaptation |
|--------|-------------------|-------------------|
| Total instruction | 6 hours (9:00-4:00) | 5 hours across 2 sessions |
| Module length | 90 min | 45 min |
| Break frequency | Every 90 min | Every 45 min |
| Group work | Table groups of 4-5 | Breakout rooms of 3-4 |
| Materials | Printed workbooks | Shared Google Doc + PDF download |
| Whiteboarding | Physical sticky notes on wall | Miro board with digital stickies |
| Buffer time | 10% | 15% |

### Module 1: Why Agile? (9:15-10:00, 45 min)

**Objective:** Explain the core principles of Agile and contrast with traditional project management.

| Phase | Duration | Activity | Platform Feature | Tech Backup |
|-------|----------|----------|-----------------|-------------|

| Instruction | 8 min | Waterfall vs. Agile comparison with animated diagram | Screen share + annotation | Static image in shared doc |
| Practice | 12 min | Breakout rooms: "Identify 3 waterfall patterns in your team's current process" | Breakout rooms (groups of 3) | Main room discussion with hand-raising |
| Debrief | 7 min | Each group's reporter shares 1 key finding | Gallery view + unmute | Type findings in chat |

**Virtual Facilitation Notes:**
- Cap instruction at 10 min before interaction
- Have pre-built Miro board sections labeled per group
- Visit each breakout room at least once (3 rooms Ã— 2 min visits)
- If breakout rooms fail: run as whole-group discussion, use chat for parallel input

### Module 4: Sprint Planning Simulation (13:40-14:25, 45 min)

**Objective:** Conduct a Sprint Planning session given a product backlog and team capacity.

| Phase | Duration | Activity | Platform Feature | Tech Backup |
|-------|----------|----------|-----------------|-------------|
| Setup | 5 min | Explain simulation rules, share backlog in Miro | Screen share Miro | PDF backlog in chat |
| Simulation | 25 min | Breakout teams: Plan a 2-week sprint from provided backlog using Miro sprint board | Breakout rooms + Miro | Google Doc sprint template |
| Report-out | 10 min | Each team shares their sprint commitment (facilitator screen-shares each Miro board) | Screen share | Chat summary |
| Debrief | 5 min | "What trade-offs did you make? What was surprising?" | Chat waterfall then verbal | Verbal only |

**Differentiation:**
- **Floor:** Complete sprint planning using provided template with all items pre-written
- **Scaffold:** Same task with a worked example sprint plan to reference
- **Extension:** Add capacity constraints (one team member at 50%) and re-plan

## Key Adaptation Decisions

| Decision | Rationale |
|----------|-----------|
| Split into 2 sessions with 90-min lunch | Reduces screen fatigue; allows catch-up |
| 45-min modules instead of 90-min | Maintains attention in virtual format |
| Miro for all group activities | Consistent tool reduces cognitive load |
| Google Docs for individual work | Allows facilitator to monitor progress in real-time |
| Chat waterfall for sharing | Equalizes participation; introverts contribute more easily |
| Breakout pairs for peer review | Shorter and more intimate than group breakouts |
| Pre-built templates for all activities | Eliminates setup time during modules; reduces tech confusion |
